## 6. GitHub Documentation Standard

Every important experiment should contain:

1. Objective
2. Environment
3. Commands
4. Expected result
5. Actual result
6. Problems encountered
7. Troubleshooting process
8. Final conclusion
9. Follow-up improvement

Recommended commit prefixes:

```text
docs: documentation update
lab: laboratory experiment
feat: new script or capability
fix: problem correction
refactor: structure improvement
chore: repository maintenance
```

---

## 7. Security Rules

Never upload:

- Passwords
- API tokens
- Private keys
- Company internal documents
- Real production addresses
- Sensitive IP addresses
- Paid course files
- Personal identity documents

Use placeholders such as:

```text
<SERVER_IP>
<USERNAME>
<PASSWORD>
<API_TOKEN>
```

## 8. Progress Definition

### Completed

A topic is completed only when:

- The course content has been studied
- The experiment has been performed
- The result has been verified
- The notes have been written
- The content has been committed to GitHub

### Not Completed

The following does not count as completion:

- Only watching a video
- Only copying commands
- Running commands without understanding the result
- Writing notes without hands-on practice
